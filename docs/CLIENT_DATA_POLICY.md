# Client Data Policy

## Purpose

The GitHub Pages workspace is a public status and coordination surface. It is not a client file repository.

## Allowed public content

- Sanitized project name or internal code
- Current stage, milestones, owners, and dates
- Nonconfidential drafting standards approved for publication
- File categories and review status without private links or sensitive filenames
- Public-safe commercial placeholders such as `TBD` or `Awaiting estimate`
- Sanitized update summaries and next actions
- Automation descriptions that do not disclose client data

## Prohibited public content

- Native CAD/BIM files, scans, point clouds, private PDFs, or project photographs
- Exact addresses unless BDPC explicitly authorizes publication
- Private cloud links, credentials, tokens, account identifiers, or access instructions
- Client correspondence, private notes, pricing that is not approved for publication, or legal documents
- Screenshots containing drawings, file paths, inbox content, contact details, or private metadata
- Automated reports containing proprietary geometry or source-file content

## Storage boundary

Private source materials remain in the approved private transfer location. Local browser edits remain on the current device until exported. Exported JSON should be reviewed before sharing or committing.

## Publication review

Before every deployment:

1. Search for addresses, email addresses, phone numbers, private URLs, tokens, and filenames.
2. Confirm no source files or screenshots were added.
3. Confirm commercial values are approved for public display.
4. Confirm all status statements are verified and current.
