# Operating Model

## Priority order

1. Complete the client work.
2. Protect accuracy and professional boundaries.
3. Minimize BDPC review time.
4. Capture reusable standards.
5. Automate repeatable inspection, reporting, and packaging.
6. Experiment only when reversible and non-disruptive.

## Status discipline

Every tracked item uses a visible status and owner. `Received`, `Opened`, `Reviewed`, and `Usable` are separate signals.

## Client communication

- Consolidate questions.
- State the consequence of each unresolved issue.
- Offer a recommended option.
- Avoid fragmented check-ins.
- Never silently change scope, fee, schedule, or acceptance criteria.

## No-database collaboration

The published GitHub version is the shared baseline. Browser edits save locally and can be exported as JSON. Local edits are not synchronized between devices.

## Commercial progression

1. Complimentary receipt and openability confirmation
2. Paid diagnostic when substantial investigation is required
3. Fixed-fee bounded pilot
4. Repeatable per-project or reserved-capacity delivery
5. Separate standards and automation support after production trust is established
